# CS-465-Full-Stack-Development

Architecture: 

  The differing methonds used in fullstack development in this course each specialized in different aspects of the application.  Express worked primarily with backend HTML files, with Javascript and SPA working to 
  streamline the process of moving between pages, and managing assets for the user's view.  The backend used NoSQL MongoDB as a database because sql is a language specifically designed to work well with databases, with mongo
  being one of the most common to be used.  These tools made it easier to manage stored assets and information within the application database, as well as making it easier to manage the database.

Functionality:

  JSON connects the front and backend development by essentially translating between the two.  JSON operates in data format, which is the specific format for files to be used when sending a query to the backend, or 
  retrieving data from the front end.  One specific instance with this applied within the application would be the application of RESTFUL API, which moves away from hard-coded database queries and into a more dynamic approach.
  This makes it easier for scaling up the application and providing modular services that may be applied in multiple scenarios.

Testing: 

  Using Windows Powershell as well as localhost were the primary methods used for testing the application's functionality.  Additionally, RESTFUL endpoints were utilized to provide logical endpoints for the user, which could 
  be debugged individually which would make the debugging process easier and faster.  Additional layers of security made testing somewhat more difficult due to the additional security measures required to access the content, including
  methods to send and retrieve security credentials to access certain pages and assets.  

Refletion: 

  This course helped me to get a better understanding of how the workflow for a fullstack application would operate outside of my education.  I feel more confident in being able to construct a web application, and most importantly, what 
  to look for to figure out how to solve any issues I may encounter when working on a similar project.  I have developed my skills working with databases and html webpages, gaining experience in what to look for when debugging and updating
  web applications.
